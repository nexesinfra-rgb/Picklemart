# SHA Keys Setup for Firebase

This guide explains how to extract SHA-1 and SHA-256 keys from your Android keystore files for Firebase configuration.

## Why SHA Keys Are Needed

Firebase requires SHA-1 and SHA-256 keys to:
- Enable Google Sign-In
- Verify app authenticity
- Enable App Check (if configured)

You need to add **both debug and release** SHA keys to Firebase Console.

## Prerequisites

- Java JDK installed (for `keytool` command)
- Keystore file(s) available

## Step 1: Get Debug SHA Keys

The debug keystore is automatically generated by Flutter/Android Studio and is located at:

**Windows:**
```
C:\Users\<YourUsername>\.android\debug.keystore
```

**macOS/Linux:**
```
~/.android/debug.keystore
```

**Default password:** `android`

### Extract Debug SHA Keys

Run this command (replace `<YourUsername>` with your actual username on Windows):

**Windows (PowerShell):**
```powershell
keytool -list -v -keystore "$env:USERPROFILE\.android\debug.keystore" -alias androiddebugkey -storepass android -keypass android
```

**macOS/Linux:**
```bash
keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android
```

Look for:
- **SHA1:** (a 40-character hex string)
- **SHA256:** (a 64-character hex string)

## Step 2: Get Release SHA Keys

For release builds, you need to extract SHA keys from your release keystore.

### If You Have the Keystore File

**Windows (PowerShell):**
```powershell
cd android
keytool -list -v -keystore app/upload-keystore.jks -alias upload
```

**macOS/Linux:**
```bash
cd android
keytool -list -v -keystore app/upload-keystore.jks -alias upload
```

You'll be prompted for the keystore password (from `key.properties` file).

### If You Don't Have the Keystore Yet

1. Generate the keystore first (see `android/KEYSTORE_SETUP.md`)
2. Then extract SHA keys using the command above

## Step 3: Add SHA Keys to Firebase

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Select your project
3. Go to **Project Settings** (gear icon)
4. Scroll down to **Your apps** section
5. Click on your Android app (or add one if not exists)
6. Click **Add fingerprint**
7. Paste the SHA-1 key
8. Click **Add fingerprint** again
9. Paste the SHA-256 key
10. Click **Save**

## Step 4: Download Updated google-services.json

After adding SHA keys:

1. Download the updated `google-services.json` file from Firebase Console
2. Replace the existing file at: `firbase apis/google-services.json`
3. Rebuild your app

## Verification

After adding SHA keys and updating `google-services.json`, verify:

1. Build a release AAB: `flutter build appbundle --release`
2. Test Google Sign-In (if implemented)
3. Check Firebase Console for any errors

## Troubleshooting

### "SHA key not found" error

- Make sure you added **both** debug and release SHA keys
- Verify the keystore alias is correct (`androiddebugkey` for debug, `upload` for release)
- Ensure you're using the correct keystore file

### "Invalid SHA key" error

- Double-check you copied the entire SHA key (no spaces, no line breaks)
- Verify the key format (SHA-1: 40 chars, SHA-256: 64 chars)

### Can't find debug.keystore

- It's created automatically when you first build an Android app
- If missing, create it manually or build the app once in debug mode

## Important Notes

- **Never commit** keystore files or `key.properties` to version control
- **Backup your release keystore** - losing it means you can't update your app on Google Play
- SHA keys are **not sensitive** - they're used for verification, not authentication
- You can add multiple SHA keys to Firebase (useful for team development)

## Quick Reference Commands

### Debug SHA Keys (One-liner)
**Windows:**
```powershell
keytool -list -v -keystore "$env:USERPROFILE\.android\debug.keystore" -alias androiddebugkey -storepass android -keypass android | Select-String "SHA"
```

**macOS/Linux:**
```bash
keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android | grep SHA
```

### Release SHA Keys (One-liner)
**Windows:**
```powershell
cd android; keytool -list -v -keystore app/upload-keystore.jks -alias upload | Select-String "SHA"
```

**macOS/Linux:**
```bash
cd android && keytool -list -v -keystore app/upload-keystore.jks -alias upload | grep SHA
```

(You'll need to enter the keystore password when prompted)
